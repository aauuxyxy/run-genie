# RunGenie - Blueprint (設計・実装指示書)

## 1. アプリ概要 (要件定義)
マラソンを趣味とするランナー向けに、指定された距離で自動的にランニングコースを生成するスマートフォンアプリ。

- **コア機能**:
  - ユーザーの現在地と「目標距離」に基づき、周回または折り返しコースを生成。
  - ルートのマップ描画と、コースの高低差（標高データ）の表示。

## 2. 技術スタック (アーキテクチャ設計)
### フロントエンド (モバイルアプリ)
- **React Native (Expo)**: クロスプラットフォーム開発。
- **react-native-maps**: 地図表示およびポリライン描画。
- **UIライブラリ**: React Native Paper 等。

### バックエンド (APIサーバー)
- **Node.js (NestJS または Express)**: コース生成のロジックや外部APIとの通信をカプセル化。
- **地図API**: Google Maps API または Mapbox API (Directions API, Elevation API)。
- **デプロイ・DB**: 必要に応じて Firebase や Vercel/Render 等を利用。
